STEP 2: CREATE A GITHUB REPOSITORY
═══════════════════════════════════════════════════════════════════════════════

Cloudflare Pages deploys from GitHub. You need a GitHub account (free).

1. Go to https://github.com/new
2. Create a new repository:
   Repository name:  gaulos-website
   Description:      Gaulos — The vessel between markets
   Public:           Yes (recommended for simplicity)
   DO NOT initialize with README/license yet
   Click "Create repository"

3. You'll see instructions. Follow "…or push an existing repository":

4. On your computer, in the /outputs folder, run these commands in Terminal/Command Prompt:

   git init
   git add .
   git commit -m "Initial commit: Gaulos website"
   git branch -M main
   git remote add origin https://github.com/YOUR_USERNAME/gaulos-website.git
   git push -u origin main

   (Replace YOUR_USERNAME with your actual GitHub username)

5. Refresh github.com. You should see all your files in the repository.

═══════════════════════════════════════════════════════════════════════════════
STEP 3: CONNECT CLOUDFLARE PAGES
═══════════════════════════════════════════════════════════════════════════════

1. Go to https://pages.cloudflare.com (sign in or create a Cloudflare account)

2. Click "Create a project" → "Connect to Git"

3. Authorize Cloudflare to access your GitHub account
   (You'll be redirected to GitHub to approve)

4. Select the "gaulos-website" repository

5. Configure build settings:
   Project name:              gaulos-website
   Production branch:         main
   Build command:             (leave EMPTY — no build needed)
   Build output directory:    /  (just the root, or leave blank)

6. Click "Save and Deploy"

Cloudflare will now deploy your site. Wait 2-5 minutes for the build to complete.

═══════════════════════════════════════════════════════════════════════════════
STEP 5: CONNECT YOUR CUSTOM DOMAIN
═══════════════════════════════════════════════════════════════════════════════

To use gaulos.eu instead of gaulos-website.pages.dev:

1. In Cloudflare Pages, select your "gaulos-website" project

2. Click "Custom domains" in the left sidebar

3. Click "Set up custom domain"

4. Enter: gaulos.eu

5. Cloudflare will check your domain registrar. Two options:

   OPTION A (Easiest): Change nameservers to Cloudflare
   - Cloudflare will provide 2 nameserver addresses
   - Go to your domain registrar (GoDaddy, Namecheap, etc.)
   - Replace the current nameservers with Cloudflare's
   - Wait 24 hours for DNS to propagate
   - Your site will be live at gaulos.eu

   OPTION B: Add DNS records manually
   - If you're keeping your current registrar/nameservers
   - Add CNAME record: gaulos-website.pages.dev
   - Follow Cloudflare's specific instructions
   - Wait for DNS propagation

6. Test: gaulos.eu should now load your website

═══════════════════════════════════════════════════════════════════════════════
STEP 6: OPTIONAL — ENABLE CLOUDFLARE'S WEB OPTIMIZATION
═══════════════════════════════════════════════════════════════════════════════

In Cloudflare dashboard:
1. Select your domain (gaulos.eu)
2. Go to "Speed" → "Optimization"
3. Enable:
   ✓ Minify JavaScript
   ✓ Minify CSS
   ✓ Minify HTML
   ✓ Auto Minify (if available)
4. Go to "Caching" → set Browser Cache TTL to "1 month"

This improves page load speeds.

TROUBLESHOOTING
═══════════════════════════════════════════════════════════════════════════════

"Pages says 'Build failed'"
→ Check the build logs in Cloudflare Pages dashboard
→ Most common issue: incorrect folder structure
→ Verify /opportunity/index.html, /contact/index.html exist

"Links are broken / 404 errors on subpages"
→ Verify folder names match exactly:
  opportunity (not Opportunity)
  why-gaulos (not why_gaulos or "why gaulos")
  All must be lowercase with hyphens
→ Each folder MUST contain index.html

"CSS not loading / page looks broken"
→ Check browser console (F12) for 404 errors
→ Verify css/styles.css exists
→ Check HTML files reference: <link rel="stylesheet" href="/css/styles.css">

"Language toggle or chat button not working"
→ Verify js/main.js exists and is referenced in HTML:
  <script src="/js/main.js"></script>
→ Check browser console for JavaScript errors

"Site still shows old content after I updated files"
→ Hard refresh browser: Ctrl+Shift+R (Windows) or Cmd+Shift+R (Mac)
→ Or clear browser cache
→ Wait 5 minutes for Cloudflare cache to clear
